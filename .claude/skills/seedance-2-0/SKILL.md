# Seedance 2.0 — Live Video Generation

This is an **executable tool skill**, not a prompt generator. It runs a bundled Python program that calls the **real** GMI Cloud `seedance-2-0-260128` API, then downloads the produced video(s) and thumbnail(s). By default each run produces **two variants** (`--count 2`) at **720p**.

## ⚠️ Activation guard — model name ONLY

Use this skill **only** when the user explicitly names the model — `seedance 2` / `seedance2` / `seedance 2.0` / `seedance-2-0[-260128]`.

This project contains MANY skills that *write video prompts*. Generic wording — 影片生成 / 文生影 / 圖生影 / 動態生成 / 動態提示詞 / video prompt / t2v / i2v — does **NOT** belong here; let the relevant prompt-writing skill handle those (a bare "Seedance" mention as a prompt *target* also belongs there). If no explicit model name from the list appears, **do not activate**. `seedance 2 fast` / `seedance 2.0 fast` is a different model (`seedance-2-0-fast-260128`) — also do not activate.

## Workflow

1. **Confirm activation** — a model name from the list is present. If not, stop.
2. **Identify the task** (mode is chosen automatically from the materials given):
   - **Text-to-video (T2V)** — only a prompt.
   - **Reference-to-video (R2V)** — prompt + image(s). **This is the DEFAULT whenever the user provides/uploads images** without saying otherwise: pass them as `--ref` / `--ref-file` (→ `reference_images`, subject/style reference).
   - **First/last-frame (I2V)** — only when the user explicitly asks for 首幀/尾幀/首尾幀 anchoring (the video starts from / ends on the exact image): use `--first-frame(-file)` / `--last-frame(-file)`.
   - Reference video / audio / GMI asset IDs are also supported: `--ref-video(-file)`, `--ref-audio(-file)`, `--ref-asset-id`.
3. **Gather inputs:**
   - `prompt` (required). If the user gives a rough idea, you may compose a strong English motion/camera prompt for them.
   - Materials (optional): public URL → `--ref <url>` etc.; **local file** → `--ref-file <path>` etc. The script auto-uploads local files to a temp host (tmpfiles.org, ~1 h auto-delete) and uses the resulting URL. Bare filenames are looked up in `uploads/`. Guards: images ≤ 10 MB (PNG/JPEG/WebP/HEIC/HEIF, ≤ 9), videos ≤ 50 MB, audio ≤ 20 MB. (GMI accepts **public URLs only** — no base64/local upload to the API itself.)
4. **Parameters** — apply defaults; ask the user **only** for a genuinely undecided necessary value:
   - `--count` (default **2**) — unless the user explicitly asks for a specific number, keep the default: every run delivers TWO variants to choose from. Each video is a separate API request submitted in parallel (max 4 — video is expensive).
   - `--resolution` `480p`/`720p`/`1080p` (default **720p**)
   - `--duration` 4–15 seconds (default **5**)
   - `--ratio` `16:9`/`4:3`/`1:1`/`3:4`/`9:16`/`21:9`/`adaptive` — when omitted the script picks **`adaptive`** if any visual reference is given (follows the source material), else **`16:9`**.
   - Audio is generated by default; pass `--no-audio` to disable. `--seed`, `--watermark`, `--web-search` exist for special cases.
5. **Run** from this skill folder (the script resolves key/uploads/outputs relative to its own location). Video takes minutes — set a generous Bash timeout (≥ 10 min):
   ```bash
   cd .claude/skills/seedance-2-0
   # text-to-video
   python3 generate_video.py --prompt "A thunderstorm rolls over a medieval castle at night, lightning illuminating stone towers"
   # reference-to-video with a local image (auto-uploaded; DEFAULT mode for user images)
   python3 generate_video.py --ref-file uploads/character.png \
     --prompt "The character walks through a neon-lit alley, cinematic tracking shot"
   # first/last-frame anchoring (only when explicitly requested)
   python3 generate_video.py --first-frame-file uploads/bud.jpg --last-frame-file uploads/bloom.jpg \
     --prompt "The flower slowly blooms, petals unfurling in morning light"
   # single video only (when the user explicitly asks for one)
   python3 generate_video.py --count 1 --prompt "..."
   ```
   For long/complex prompts, write the prompt to a temp file and pass `--prompt "$(cat /tmp/p.txt)"` to avoid shell-quoting issues. Use `--dry-run` to preview the payload without spending quota.
6. **Show the results** — the script prints each saved `.mp4` path and a `_thumb.jpg` per video under `outputs/`. Open each **thumbnail** with the Read tool so the user gets a visual preview, and give the full `.mp4` paths (videos can't be previewed inline — the user opens them in Finder/QuickTime).

## Notes

- **API key** precedence: `--api-key` > `$GMI_API_KEY` > `.gmi_api_key` > `../gemini-3-pro-image/.gmi_api_key` (same GMI Cloud key for the whole platform). The key file is already present and gitignored.
- **Privacy** — local-file flags upload to a public temp host (~1 h then auto-deleted). For sensitive material, ask the user for a private URL and use the URL flags instead.
- **Cost** — video is billed per generated second; keep `--count`/`--duration` modest unless asked.
- Full request/response schema, status values, and limits: **`references/gmi-api.md`**.
